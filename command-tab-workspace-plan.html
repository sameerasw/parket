План реализации Command+Tab workspace follow

Цель: если macOS через Command+Tab активирует приложение, окно которого находится на другом эмулированном workspace parket, parket должен открыть нужный workspace и сфокусировать это окно.

Контекст проекта

parket не использует реальные Spaces macOS. Workspaces эмулируются массивами TrackedWindow, а переключение workspace означает: окна старого workspace уводятся offscreen, окна нового workspace раскладываются через tiler.

Из-за этого Command+Tab сейчас может активировать приложение, окно которого фактически остается offscreen. Для пользователя это выглядит как выбранное приложение без видимого окна.

Правильная точка интеграции - не перехват Command+Tab. Command+Tab должен остаться системным app switcher. parket должен реагировать на смену active application и focused window.

Текущие точки расширения

WindowObserver.swift

Уже слушает запуск, завершение, создание и уничтожение окон. Сюда нужно добавить реакцию на activation/focus.

WindowManager.swift

Уже умеет читать AXFocusedUIElement, затем AXFocusedWindow. Нужно добавить вариант для конкретного pid.

Workspace.swift

Управляет мониторами и workspace. Здесь должен появиться метод reveal для externally focused window.

Monitor.swift

Уже умеет switchTo, retile и restoreFocusedWindow. Нужен вариант switch с фокусом конкретного окна.

Архитектурное решение

  1. Добавить в WindowManager метод focusedWindow(pid: pid_t) -> TrackedWindow?.
  2. Текущий focusedWindow() оставить как wrapper: взять NSWorkspace.shared.frontmostApplication и вызвать pid-specific метод.
  3. Добавить в WorkspaceManager метод followExternalFocus(pid: pid_t). Все мутации состояния должны выполняться через main queue.
  4. Реализовать поиск окна по всем мониторам и workspace. Результат должен содержать monitorIndex, workspaceIndex, windowIndex и сохраненный TrackedWindow.
  5. Если окно найдено на inactive workspace, переключить этот monitor на нужный workspace, разложить окна и сфокусировать найденное окно.
  6. Если окно уже на active workspace, не делать лишний retile. Достаточно обновить focused monitor, focused index и status bar.

Алгоритм обработки

  1. Пользователь нажимает Command+Tab и выбирает приложение.
  2. macOS делает выбранное приложение frontmost.
  3. parket получает NSWorkspace.didActivateApplicationNotification или AX focus notification.
  4. Через короткий debounce parket читает focused window выбранного pid.
  5. Если focused window не tileable или не tracked, parket ничего не меняет.
  6. Если focused window tracked и находится на inactive workspace, parket раскрывает этот workspace.
  7. После retile parket вызывает TrackedWindow.focus() на сохраненном tracked instance, чтобы сохранить корректную работу native tabs.

Изменения по файлам

Sources/WindowManager.swift

  • Добавить static func focusedWindow(pid: pid_t) -> TrackedWindow?.
  • Внутри использовать AXUIElementCreateApplication(pid).
  • Сохранять порядок чтения: сначала AXFocusedUIElement, затем AXFocusedWindow.
  • Оставить strict filtering через isTileable.

Sources/Workspace.swift

  • Добавить приватный тип WindowLocation.
  • Добавить locateWindow(_ window: TrackedWindow) -> WindowLocation?.
  • Добавить followExternalFocus(pid: pid_t).
  • Не синхронизировать полный snapshot pid внутри focus-follow. Hidden окна одного приложения могут иметь одинаковую offscreen позицию и ошибочно сгруппироваться как native tabs. Full snapshot sync остается в observer path для create/destroy событий.
  • При переходе на другой monitor обновлять focusedMonitorIndex до вызова StatusBar.shared.update().

Sources/Monitor.swift

  • Добавить метод revealWorkspace(_ index: Int, focusing focused: TrackedWindow).
  • Если index != active, сохранить focused index старого workspace, обновить previousActive и active, спрятать окна старого workspace offscreen.
  • Установить focusedIndices[active] в windowIndex.
  • Вызвать retile(), затем сфокусировать окно.
  • Для monocle после focus вызвать raise(), как в текущей логике.

Sources/WindowObserver.swift

  • Добавить подписку на NSWorkspace.didActivateApplicationNotification.
  • В observeApp(pid:) добавить app-level AX notifications: kAXFocusedWindowChangedNotification и kAXFocusedUIElementChangedNotification.
  • В AX callback для этих notifications вызывать WorkspaceManager.shared.followExternalFocus(pid:).
  • Не добавлять CoreGraphics identity и не использовать private APIs.

Sources/Hotkeys.swift

  • Не перехватывать Command+Tab как новый hotkey.
  • Проверить конфликт с modifier = "command" и текущим last_workspace = "tab". Рекомендуемое поведение: системный Command+Tab должен проходить в macOS, а parket follow должен сработать после activation.

Debounce и защита от гонок

  • Добавить debounce 30-80 ms, потому что AX focused window может обновиться позже, чем NSWorkspace activation.
  • Сделать 2-3 retry с коротким интервалом, если focused window еще не читается.
  • Добавить простой guard вроде isFollowingExternalFocus, чтобы focus, вызванный самим parket, не запускал повторный reveal loop.
  • Если focused AX element не совпал с tracked window, fallback можно делать только когда у pid есть ровно одно tracked tileable window. Иначе лучше ничего не менять, чтобы не раскрыть неверный workspace.

Поведение на нескольких мониторах

  • Поиск окна должен идти по всем WorkspaceManager.monitors.
  • Если окно найдено на другом monitor, нужно обновить focusedMonitorIndex.
  • Переключать workspace нужно только на monitor, где находится окно.
  • Остальные мониторы не должны менять active workspace.

Native tabs

  • Не использовать CGWindowList, native window IDs или AXTabs как identity.
  • Сравнение должно идти через существующий TrackedWindow, members, group и focusElement.
  • При раскрытии workspace фокусировать saved tracked instance, а не создавать новый объект без members. Это важно для Ghostty и других приложений с native tabs.

Проверка

  • Unit/build: make check.
  • Manual: Terminal на workspace 1, Safari на workspace 2. Находясь на workspace 1, выбрать Safari через Command+Tab. Должен открыться workspace 2.
  • Manual: одно приложение с окнами на двух workspace. Проверить, что parket раскрывает именно workspace сфокусированного окна.
  • Manual: Ghostty native tabs. Command+Tab не должен разделять tabs на отдельные tileable окна и не должен менять active tab.
  • Manual: два монитора. Command+Tab к окну на другом monitor должен обновить focused monitor indicator и открыть workspace только на нужном monitor.
  • Manual: dialogs, panels, fullscreen и minimized окна не должны попадать в reveal/tiling.

Definition of Done

  • Command+Tab к offscreen tracked window открывает правильный workspace.
  • Окна на текущем workspace не вызывают лишний retile или заметный flicker.
  • Native tabs продолжают группироваться через members.
  • Нет private macOS APIs.
  • Нет новых dependencies.
  • Нет CoreGraphics identity для tiling.
  • make check проходит.
